Your Claude Code brain, on every machine.

The finding

The memory folder is named after the machine's home path.

Claude Code stores auto-memory under ~/.claude/projects/<encoded-home>/memory/. That encoded name is different on every machine:

  • macOS-Users-alice
  • WindowsC--Users-bob
  • Linux-home-dev

Every "sync your Claude config" tool copies the files verbatim — and inherits this bug. braingraft doctor is the only thing that tells you it's happening to you.

The fix, in three moves

Diagnose it. Pack it. Graft it — reversibly.

Each step is a single command. The first two never write to your system. The third is a dry-run until you say otherwise.

Diagnose 01
braingraft doctor

Read-only. Finds memory Claude can't see, classifies every path, flags secrets and non-portable absolute paths. Cannot break anything.

● INVISIBLE  61 KB
brain  142 files
▲ 3 absolute paths
exit 2 · findings
Pack 02
braingraft pack -o brain.brain

A portable, structurally-sanitized bundle. Home paths tokenized, per-entry sha256. Refuses to build if it finds a secret.

✓ allowlisted   142
✓ path-tokenized
✓ sha256 per entry
→ brain.brain 1.1 MB
Graft 03
braingraft graft --apply

Dry-run by default. Memory re-encoded for this machine's home path. Staged, journaled, and undoable in one command.

＋ memory  re-encoded
＝ settings merged
· mcp   → pending
undo restores exactly
doctor→pack→inspect→graft --apply→undo (if you change your mind)
What it never touches

Your secrets and your conversations stay on your machine.

The bundle is an allowlist, not a denylist. If it isn't explicitly portable, it never leaves.

Credentials are unpackable

.credentials.json, tokens and API keys are structurally dropped. No flag overrides it.

Transcripts never leave

Your conversation history is never read, packed, or synced. The tool only moves the brain — skills, agents, memory.

Nothing phones home

No network calls, ever. No analytics, no accounts, no telemetry. It runs, then exits.

Zero dependencies

Node stdlib only — zlib, crypto, fs. Plain ESM, no build step. Auditable in an afternoon.

Zero accounts

Nothing to sign up for. It's a CLI you run on your own files — the whole thing works offline.

MIT licensed

Free, open, forkable. The name is a single constant — the whole surface is yours to read and change.

Safe by construction

Grafting writes code Claude will run. So a bundle is treated as untrusted.

Hooks execute, skills steer the model, MCP servers launch subprocesses. Consent is computed from the actual bytes — never from what the bundle claims about itself.

Default

Dry-run first

Every graft prints a per-file verdict table and stops. Nothing is written until you add --apply.

Reversible

Snapshot & undo

Writes are staged and journaled. braingraft undo restores the previous state byte-for-byte.

Reviewed

Untrusted by default

Every executable and instruction file is listed before apply. A foreign bundle requires an explicit --trust. MCP servers never auto-enable.

Optional

AES-256-GCM

--encrypt for the memory-on-a-USB-stick case. scrypt-derived key, tag verified before a single byte is written.
